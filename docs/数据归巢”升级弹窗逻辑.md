**“数据归巢”升级弹窗逻辑**：

---

### 🚀 第一阶段：身份觉醒 (Identity Shift)

当用户支付成功，从 `ls-web` 首次进入 `jss-web` 时，我们不直接显示空白看板，而是弹出一个半透明的“建筑蓝”蒙层。

- **欢迎语**： “欢迎来到项目管理时代，CEO Patrick。检测到你手头有 **128 张** 尚未归类的收据。让我们把它们变成利润看板。”

- **核心动作**： **“创建我的第一个项目”** (活力橙按钮)。

---

### 🏗️ 第二阶段：建立“阵地” (First Project Wizard)

这一步要极其简单，只需两个字段：

1. **项目昵称**： (例如：Burnaby Kitchen Renovation)。

2. **工地地址**： (用于开启我们讨论过的 **GPS 考勤校验**)。

---

### 📦 第三阶段：数据“大迁徙” (Data Harvesting)

这是最让用户尖叫（Aha Moment）的一步。利用我们存储在 `JSONB` 里的灵活性，我们进行“数据钓鱼”：

- **逻辑**： 系统自动检索所有 `project_id` 为空的收据。

- **交互**： 弹出一个卡片流，显示最近的 5 张收据缩略图（如：Home Depot 的木材、RONA 的油漆）。

- **指令**： “这些收据是属于 [Burnaby Kitchen] 的吗？”

- **选项**：
  
  - [ 全选并关联 ] —— 瞬间，这些死数据就变成了该项目的 **“已发生材料成本”**。
  
  - [ 稍后处理 ] —— 进入待办列表。

---

### 👥 第四阶段：团队激活 (Worker Invite)

既然是 JSS，必须体现协作。

- **提示**： “老板，这活儿你自己干？邀请你的工头，让他们帮你拍照。”

- **动作**： 输入一个手机号，发送一条带下载链接的短信。

---

### 📊 升级后的逻辑差异 (Summary Table)

| **升级前 (LS 视角)** | **升级后 (JSS 视角)**    |
| --------------- | ------------------- |
| **收据** = 报税凭证   | **收据** = 项目材料成本     |
| **金额** = 个人支出   | **金额** = 消耗的项目预算    |
| **操作人** = 仅自己   | **操作人** = 老板核账，工人拍照 |
