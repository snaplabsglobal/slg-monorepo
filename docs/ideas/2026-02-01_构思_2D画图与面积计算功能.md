# 2D 画图与面积计算功能构思

> **文档类型：** 产品构思  
> **提出人：** CEO  
> **讨论对象：** GPT (CPO)  
> **日期：** 2026-02-01  
> **状态：** 💡 构思阶段（未决策）

---

## 📌 背景

在 JSS / LedgerSnap 项目中，发现会计在处理施工项目时，经常需要：

- 计算施工面积
- 统计材料数量
- 核对图纸标注

目前这些工作需要手动完成，效率低且容易出错。

---

## 💡 核心想法

### 近期构想：在 JSS 中加入 2D 画图功能

**基础功能：**

- 画直线
- 添加标注
- 自动计算面积
- 自动统计数量

**使用场景：**

- 会计在处理施工账单时，快速绘制施工区域图
- 自动计算面积，匹配账单中的平方英尺数
- 标注材料使用位置，方便核对

### 远期展望：水管工专用 Isometric 绘图软件

**目标用户：** 水管工、HVAC 技术人员

**核心功能：**

- 专业的 Isometric 视图绘制
- 管道、接头、阀门等预设组件库
- 自动生成材料清单（BOM）
- 与 JSS 账务系统集成

**市场定位：**

- 比 AutoCAD 简单易用（专门针对水管工）
- 比手绘更专业准确
- 与 LedgerSnap 数据联动（画完图直接生成采购单）

---

## 🗣️ 与 GPT (CPO) 的讨论要点

### GPT 提出的关键建议：

1. **MVP 定义：** 先做最简单的直线 + 面积计算，不要一开始就做复杂的 Isometric
2. **会计接受度：** 会计不需要很精美的图，能算对面积就行
3. **技术风险：** 2D 绘图相对简单，但 Isometric 需要 3D 渲染库，复杂度高
4. **产品差异化：** 如果只是通用 2D 画图，市场上已有很多工具；水管工专用 Isometric 是差异化点

### 需要进一步探讨的问题：

- [ ] 会计实际工作中多久需要画一次图？
- [ ] 是否可以让他们用现有工具（如 Google Drawing）然后导入数据？
- [ ] Isometric 绘图的市场规模有多大？
- [ ] 技术实现难度（需要 CTO 评估）

## 🎯 可能的应用场景

### 场景 1：施工面积计算

1. 会计收到一张施工账单，写着"安装地板 500 sqft"
2. 在 LedgerSnap 中打开画图工具
3. 快速画出房间轮廓
4. 系统自动计算面积 = 498 sqft
5. 标注偏差原因，备注在账单中

### 场景 2：材料数量核对

1. 账单列出"管道 20 根，接头 15 个"
2. 在图上标注管道位置
3. 系统自动统计：实际需要管道 18 根，接头 14 个
4. 发现账单可能多收费，提醒会计核实

### 场景 3：水管工绘图（远期）

1. 水管工接到项目，需要设计管道布局
2. 在专用软件中快速绘制 Isometric 视图
3. 系统自动生成材料清单
4. 一键同步到 LedgerSnap，生成采购订单

## 🔧 技术可行性初步判断

### 2D 画图功能

**难度：** ⭐⭐☆☆☆（中低）

**可能的技术方案：**

- Canvas API（原生 Web）
- Fabric.js（轻量级画布库）
- Konva.js（功能更强）

**核心挑战：**

- 面积计算算法（多边形面积公式）
- 标注与数据关联
- 移动端适配

### Isometric 绘图软件

**难度：** ⭐⭐⭐⭐☆（高）

**可能的技术方案：**

- Three.js（3D 渲染）
- Babylon.js（更强大但更复杂）
- 或者用 2D 模拟 Isometric 视角（降低难度）

**核心挑战：**

- 3D 空间计算
- 组件库设计（管道、接头等预设形状）
- 性能优化
- 需要专业水管工参与产品设计
