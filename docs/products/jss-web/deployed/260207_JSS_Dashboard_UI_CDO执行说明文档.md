## 🔒 设计冻结项（不可随意改）

### 以下内容视为产品裁决结果

---

### 🔒 1. 信息架构（冻结）

```
左侧（桌面）/ 抽屉（移动）导航结构

导航顺序：
Jobs → Camera → Self-Rescue → Settings

Camera不是子功能，是一级动作
```

**为什么冻结：**
```
这个顺序经过产品哲学讨论
Camera的一级位置体现"Snap First"
Self-Rescue的一级位置体现差异化战略
```

---

### 🔒 2. 视觉基调（冻结）

#### 主品牌色

```
JSS活力橙
对应现有实现（Tailwind amber-500）

❌ 禁止替换为其他橙色体系
```

---

#### 整体风格

```
工具感 > 装饰感

具体表现：
- 少阴影
- 少拟物
- 清晰、可靠、工程感
```

**为什么这样：**
```
JSS的用户是工地现场人员
不是设计师
不需要"好看"，需要"好用"
```

---

### 🔒 3. 角色假设（冻结）

**默认用户是：**

```
✓ 总包 / 员工
✓ 非设计师
✓ 非SaaS熟练用户
```

**设计原则：**
```
所有UI决策
向"不会思考的现场用户"倾斜

不要假设用户会：
- 探索功能
- 看文档
- 理解隐喻
```

---

### 🔒 4. Mobile First（冻结）

```
✓ 所有主操作必须在单屏可见
✓ 无横向滚动
✓ CTA固定在底部（safe area）
✓ 单拇指可完成核心流程
```

**为什么：**
```
工地现场：
- 戴手套
- 手脏
- 一只手拿材料
- 不能双手操作
```

## ⚠️ 常见错误（请避免）

### ❌ 错误1：自由发挥交互

**错误示例：**
```
"我觉得Self-Rescue可以做成弹窗"
"Camera可以放在Jobs里面"
"这个颜色我改成蓝色更专业"
```

**正确做法：**
```
如果认为设计有问题
→ 先完成当前设计
→ 再提出改进建议
→ 等待产品决策
```

---

### ❌ 错误2：技术优先于产品

**错误示例：**
```
"我用React Native重写更高效"
"我把这几个页面合并成一个"
"我把导航改成Tab更标准"
```

**正确做法：**
```
技术选型可以灵活
但产品结构必须忠实
```

---

### ❌ 错误3：移动端降级

**错误示例：**
```
"移动端我简化一下"
"这个按钮移动端先不做"
"移动端横向滚动没关系"
```

**正确做法：**
```
JSS是Mobile First
移动端不是"简化版"
移动端是"第一优先级"
```
